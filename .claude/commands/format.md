# format

Prettier と ESLint でコードをフォーマット・整形する。

## 使用シーン

- コミット前のコード品質確認
- インデント・引用符の統一
- ESLint ルール違反の自動修正

## コマンド

```bash
# Prettier で自動フォーマット
npm run format

# ESLint で修正
npm run lint

# 両方実行
npm run format && npm run lint
```

## フォーマットルール

- **引用符**: シングル (`'`)
- **行末**: セミコロン付き
- **末尾カンマ**: 常に付ける
- **行長**: 100 文字

## 確認事項

- ファイル保存時に自動フォーマットされているか
- CI/CD でフォーマット チェックが通っているか
