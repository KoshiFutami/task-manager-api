# test

テストを実行して、修正が必要な部分を確認する。

## 使用シーン

- コード変更後のテスト実行
- 機能実装前のテスト駆動開発
- カバレッジ確認

## コマンド

```bash
# 全テスト実行
npm run test

# 特定ファイルのテスト
npm run test -- tasks.service.spec

# Watch モード（開発中）
npm run test:watch

# カバレッジレポート
npm run test:cov
```

## 確認ポイント

1. すべてのテストがパスしているか
2. 新規機能のテストが追加されているか
3. カバレッジが十分か（目安: 80% 以上）
